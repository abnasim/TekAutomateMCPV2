# Procedure Template

Use this file as the starting point for new `scope_logic` entries.

## Trigger Signals

- Phrase or symptom the AI should match
- Common query wording from users
- Specific sentinels, errors, or readback values

## Procedure

1. First safe step to try.
2. Second diagnostic or configuration step.
3. Third corrective step.
4. Verification step.
5. Escalation or fallback step if needed.

## Notes

- Constraints or caveats.
- Things the AI should avoid doing automatically.
- Helpful reminders about when to stop and ask for user input.
